`mdprev` の実装品質を上げるため、既知の改善項目と仕様確定事項を TODO 化する。

## 優先タスク
- [ ] `NSEvent` ローカルモニタのライフサイクルを修正する
  - 現状: `AppModel` 初期化時に `addLocalMonitorForEvents` を登録するが、解除処理がない。
  - 対応:
    - ウィンドウ/モデル破棄時にモニタを確実に解除する。
    - 複数ウィンドウの開閉を繰り返してもモニタが増殖しないことを確認する。

- [ ] `RecentFilesStore` の読み込み時にも最大件数 (`10`) を強制する
  - 現状: `loadURLs` では既存データの上限カットをしていない。
  - 対応:
    - ロード時点で最大件数に切り詰める。
    - 重複排除・順序維持の既存仕様を壊さない。

- [ ] Recent Files クリック時のファイルタイプごとの挙動を制御する
  - 仕様:
    - `.md` / `.markdown` のみ「新規ウィンドウで開く」対象とする。
    - それ以外は仮実装として、そのファイルがあるフォルダを Finder で開く。
  - TODO:
    - 非 Markdown ファイルをどう扱うべきかの本実装方針を後で決める。

- [ ] `mdprev-open-file://` の扱いを制限する
  - 現状: カスタムスキームリンクを無条件でファイルオープン処理している。
  - 対応:
    - 上記ファイルタイプ制御を適用する。
    - 期待しないリンク入力でも安全側に倒れるようにする（無効化/無視/Finder フォールバック）。

- [ ] `Option + クリック`（Open Recent）の挙動を一意にする
  - 確定仕様:
    - `Option + クリック` は「新規ウィンドウを作って、選択したファイルを開く」。
    - 既存ウィンドウの有無に依存させない。
  - 対応:
    - フォーカスウィンドウがある場合でも、現在ウィンドウを上書きせず新規ウィンドウで開く。
    - フォーカスウィンドウがない場合も同じ挙動にする。
